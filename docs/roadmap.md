# Live Board 実装ロードマップ

## 1. 進め方

機能一覧を横断的に一度に実装せず、各フェーズで「編集操作 → 保存 → OBS表示 → テスト」まで通る縦のスライスを完成させる。

優先順位:

1. データ破損を防ぐ基盤
2. 編集ページと配信ページの分離
3. OBSへ安定表示できる最小経路
4. ページ・レイヤー・画像・描画機能
5. 配信中の操作性と性能
6. 高度編集と共同編集

## Phase 0: リポジトリ基盤

### 目的

実装を安全に分割し、CIで最低限の品質を担保する。

### 対象

- pnpm workspace
- Electron + Vite + React + TypeScript
- `apps/desktop`、`apps/overlay`、`packages/domain`の初期構成
- ESLint、TypeScript、Vitest、Playwright
- main / preload / rendererのセキュリティ設定
- GitHub Actionsでlint、typecheck、test、build
- ADR・要件・セキュリティ文書

### 完了条件

- デスクトップアプリとOverlayの空画面が起動する
- RendererにNode APIが露出していない
- CIが成功する
- READMEの手順だけでローカル起動できる

## Phase 1: Workspace / Project / Page 縦スライス

### 対象

- ワークスペース作成・名前変更・保存・再読込
- プロジェクト作成
- ページ追加・複製・削除・並び替え
- 編集ページと配信ページの別管理
- ページ一覧とサムネイルの最小UI
- OBS Overlayへ配信ページを表示
- 即時切り替えとフェード
- 自動保存の最小実装

### 完了条件

- 次ページを編集しながらOBSで現在ページを維持できる
- アプリ再起動後にページ構成が復元される
- OBS再接続時に現在の配信ページへ復帰する

## Phase 2: レイヤー基盤

### 対象

- ラスター、テキスト、画像、図形、背景、フォルダー
- 追加・削除・複製・名前変更・並び替え
- 表示・ロック・不透明度
- 通常、乗算、スクリーン、加算、オーバーレイ
- フォルダー階層と循環防止
- レイヤー単位のOBS同期
- Undo / Redo基盤

### 完了条件

- レイヤー順・表示・不透明度・合成結果がEditorとOBSで一致する
- レイヤー操作をUndo / Redoできる
- 100レイヤー時の一覧操作を計測できる

## Phase 3: 基本描画ツール

### 対象

- ペン、消しゴム、バケツ、スポイト
- Pointer Events、筆圧
- ブラシサイズ、不透明度、硬さ、間隔、手ぶれ補正
- カラーホイール、RGB / HSL / HSV / HEX
- ズーム、パン、回転、左右反転表示
- グリッド、ガイド、スナップ
- 描画履歴とメモリ上限

### 完了条件

- 1920×1080キャンバスで通常描画が60fps目標を満たす
- 筆圧対応環境と非対応環境の両方で安全に描画できる
- OBS反映時間を計測し通常100ms以内を目標に調整できる

## Phase 4: テキスト・図形・画像

### 対象

- テキスト入力、フォント、書式、縁取り、影
- 矩形・楕円等の図形
- PNG、JPEG、WebP、GIF静止画、SVGインポート
- ドラッグ＆ドロップ、クリップボード
- 移動、拡大縮小、回転、反転、トリミング
- アセット重複排除
- SVGサニタイズ、巨大画像制限

### 完了条件

- 対応形式を画像レイヤーとして安全に追加できる
- EditorとOBSでテキスト・図形・画像の表示結果が一致する
- 不正SVG、破損画像、巨大画像を安全に拒否できる

## Phase 5: 選択・変形・配信操作性

### 対象

- 矩形、楕円、投げ縄選択
- 選択範囲の移動・拡大縮小・回転
- 配信ページ切り替えショートカット
- 誤切り替え防止ロック
- ワークスペースプリセット
- OBS接続状態・遅延表示
- 黒板、ホワイトボード、透過テーマ
- カスタムCSS

### 完了条件

- 配信中の主要操作をキーボードだけでも実行できる
- 編集用選択表示がOBSへ混入しない
- カスタムCSS異常時もOverlayを復旧できる

## Phase 6: 保存・復旧・入出力の完成

### 対象

- `.liveboard`アーカイブ
- manifestとバイナリ分離
- スキーマバージョン・migration
- 自動保存ジャーナル
- クラッシュ復元UI
- ワークスペース複製、最近使用、お気に入り
- インポート時のZip Slip・圧縮爆弾対策

### 完了条件

- ワークスペース全体を別環境へ移行できる
- 保存中クラッシュ後に直前データを復旧できる
- 旧スキーマからの移行失敗時に原本が保持される

## Phase 7: 性能・長時間安定性

### 対象

- 非表示ページ・レイヤーのレンダリング抑制
- サムネイルの非同期生成
- バケツ・画像変換のWorker化
- レイヤーキャッシュ、差分レンダリング
- 100ページ、100レイヤー、4K画像の負荷試験
- 8時間相当の長時間操作試験
- メモリリーク検査

### 完了条件

- 性能基準と計測方法が文書化される
- 長時間操作で重大なメモリ増加やOBS切断が発生しない
- ボトルネックに基づきOffscreenCanvas / WebGL導入可否を判断できる

## Phase 2以降の拡張

- レイヤーマスク、クリッピングマスク
- 自動選択、色域選択、自由変形
- ブラシプリセット、先端画像、パレット管理
- パネルドッキング
- スライド、ワイプ、ズーム、CSSトランジション

## Phase 3以降の拡張

- 調整レイヤー
- メッシュ変形、液状化、高度な定規
- PSD入出力
- アニメーションページ
- LAN共同編集
- 外部サーバー共同編集

## テスト戦略

各Phaseで最低限以下を追加する。

- Domain: Unit test
- Persistence / Import: Integration test
- Editor UI: Component test
- Electron IPC: Contract test
- Editor → OBS: E2E test
- 性能: 計測スクリプトと基準値記録
- セキュリティ: 悪性fixtureによる回帰テスト

## PR分割ルール

- 1PRは1つの縦スライスまたは1つの基盤責務に限定
- UI変更と大規模内部リファクタを同一PRへ混ぜない
- 破壊的スキーマ変更はmigrationとテストを同じPRへ含める
- OBSプロトコル変更はEditor・Overlay双方と互換性テストを含める
- 性能最適化PRには変更前後の計測結果を記載する
