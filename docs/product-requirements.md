# Live Board プロダクト要件

## 1. 目的

Live Board は、配信者が配信中に素早く描画・文字入力・画像配置・ページ切り替えを行い、その結果をOBSへリアルタイム表示できるローカル完結型ペイントワークスペースである。

単一ホワイトボードではなく、複数作品・複数ページ・複数レイヤーを管理する制作環境を提供する。CLIP STUDIO PAINTの操作体験を参考にするが、全機能の再現は初期完成条件としない。

## 2. 対象ユーザー

- 雑談・ゲーム攻略・学習・イベント配信を行う配信者
- OBS上へ手書きメモ、図解、クイズ、待機画面を表示したいユーザー
- 配信しながら複数ページを切り替えたい講師・解説者
- 外部クラウドへ制作データを送信せずに利用したいユーザー

## 3. 中核価値

1. OBSへの低遅延リアルタイム表示
2. 編集ページと配信ページの分離
3. 配信中でも迷わない操作密度とショートカット
4. 背景透過オーバーレイと配信用テーマ
5. 外部サーバー不要のローカル動作
6. ワークスペース全体の保存・復元・移行
7. 将来の共同編集へ拡張できるデータ・イベント設計

## 4. データ階層

```text
Workspace
└── Project
    ├── Page
    │   ├── Layer
    │   │   └── Canvas Object / Raster Data
    │   └── Page Settings
    ├── Project Assets
    ├── Brushes / Palettes / Themes
    └── Project Settings
```

必須識別子は `workspaceId`、`projectId`、`pageId`、`layerId`、`objectId`、`assetId` とする。すべての保存データに `schemaVersion` を持たせる。

## 5. MVP機能要件

### 5.1 ワークスペース

- 新規作成、開く、名前変更、複製、削除
- 最近使用したワークスペース、お気に入り
- 自動保存、クラッシュ復元
- ワークスペース単位のインポート・エクスポート
- 複数プロジェクトの保持

### 5.2 プロジェクト・タブ

- 単一ウィンドウ内で複数プロジェクトまたはページをタブ表示
- タブ並び替え、閉じる、ピン留め、最近閉じたタブの復元
- 未保存状態の表示
- 未保存タブを閉じる際の確認

別ウィンドウ分離はMVP対象外とする。

### 5.3 ページ

- 追加、削除、複製、名前変更、並び替え
- 表示・非表示、ロック、サムネイル
- 一括選択、一括削除、一括エクスポート
- ページテンプレートから追加
- キャンバス幅・高さ、DPI、背景色・透過、グリッド、ガイド、出力範囲、セーフエリア
- 即時切り替え、フェードトランジション

### 5.4 編集ページと配信ページ

- `activeEditPageId` と `activeBroadcastPageId` を別状態として管理
- OBS表示ページの固定
- 次ページ・前ページ・番号指定ショートカット
- 切り替え前プレビュー
- 誤切り替え防止ロック
- ページ切り替えを通常100ms以内でOBSへ反映することを目標とする

### 5.5 レイヤー

MVP対象:

- ラスター、テキスト、画像、図形、背景、レイヤーフォルダー
- 追加、削除、複製、名前変更、並び替え
- 表示、編集ロック、移動ロック、透明ピクセルロック
- 不透明度、レイヤーカラー、検索、絞り込み
- 通常、乗算、スクリーン、加算、オーバーレイ
- 下のレイヤーと結合、選択レイヤー結合、表示レイヤー結合
- 破壊的操作のUndo
- 文字・図形の縁取り、ドロップシャドウ

Phase 2以降:

- ベクターレイヤー、レイヤーマスク、クリッピングマスク
- 合成モード拡充、調整レイヤー、高度なエフェクト

### 5.6 画像インポート

MVP対応形式:

- PNG、JPEG、WebP、GIF静止画、SVG

入力方法:

- ファイル選択、ドラッグ＆ドロップ、クリップボード、スクリーンショット貼り付け

編集:

- 移動、拡大縮小、回転、水平・垂直反転、トリミング
- 不透明度、合成モード、表示、ロック、複製、削除

同一画像は `assetId` 参照を基本とし、ページ・レイヤーごとにバイナリを重複保存しない。

### 5.7 キャンバス・描画

- ペン、消しゴム、バケツ、スポイト、文字入力
- ブラシサイズ、不透明度、硬さ、間隔、手ぶれ補正、入り抜き
- 筆圧によるサイズ・不透明度変化
- ズーム、パン、キャンバス回転、左右反転表示
- 全体表示、100%表示、ナビゲーター、ミニマップ
- グリッド、水平・垂直ガイド、スナップ
- 矩形・楕円・投げ縄選択
- 選択対象の移動・拡大縮小・回転
- Pointer Eventsを基盤とした筆圧・傾き・ペン種別の取得

### 5.8 履歴

- Undo / Redo
- 履歴パネル、任意履歴状態への移動
- ページ単位の履歴
- プロジェクト操作とページ内操作の区別
- 件数上限とメモリ上限

### 5.9 OBS連携

- OBS Browser Source向けローカルオーバーレイURL
- 背景透過
- 編集アプリからOBS表示状態をリアルタイム同期
- OBS接続状態表示
- 黒板、ホワイトボード、透過テーマ
- カスタムCSS
- OBS側では編集UI・選択枠・ガイド等を表示しない

## 6. 非機能要件

### 性能

- 描画中60fpsを目標
- OBS反映は通常100ms以内を目標
- 1920×1080・複数レイヤーを安定操作
- 非表示ページと非表示レイヤーを不要に再描画しない
- サムネイル、バケツ、画像変換を必要に応じて非同期化
- 長時間配信でメモリ使用量が単調増加しない

性能最適化は計測に基づき、Canvas 2D、OffscreenCanvas、Web Worker、WebGL、タイルレンダリングの順に必要性を評価する。

### 可用性・復旧

- 自動保存処理は描画を長時間ブロックしない
- 保存途中の異常終了で直前の正常データを破壊しない
- クラッシュ復元データと通常保存データを分離
- スキーマ移行に失敗した場合は元ファイルを保持する

### 保守性

- ドメインモデル、描画エンジン、永続化、OBS同期、UIを分離
- ツールは共通インターフェースで追加可能にする
- Undo可能な操作はCommandとして表現する
- 保存形式と内部状態を直接結合しない

## 7. 保存形式

JSONへ大量のBase64を埋め込まず、マニフェストとバイナリを分離したアーカイブ形式を採用する。

```text
workspace.liveboard
├── manifest.json
├── projects/
├── pages/
├── assets/
├── thumbnails/
├── brushes/
├── palettes/
├── themes/
└── styles/
```

インポート時は一時ディレクトリへ展開・検証し、成功後に正式データへ反映する。

## 8. MVP対象外

- CLIP STUDIO PAINTとの完全同等・完全互換
- PSDのレイヤー構造入出力
- 3D・アニメーションレイヤー
- メッシュ変形、液状化、高度なパース定規
- 外部サーバー共同編集
- 別ウィンドウへのタブ分離

## 9. 受け入れ条件

1. ワークスペースを作成・保存・復元できる
2. 1プロジェクトに複数ページを保持できる
3. ページを追加・複製・削除・並び替えできる
4. 編集ページとOBS表示ページを分離できる
5. OBS表示ページをショートカットで切り替えられる
6. ページごとに独立したレイヤーを保持できる
7. ラスター・文字・画像・図形レイヤーを作成できる
8. レイヤーフォルダーを作成できる
9. レイヤーの表示・ロック・不透明度・合成モードを変更できる
10. PNG、JPEG、WebP、SVGを安全に読み込める
11. 読み込んだ画像を移動・拡大縮小・回転・トリミングできる
12. SVGの危険要素・属性・外部参照を除去できる
13. キャンバスをズーム・パン・回転・左右反転表示できる
14. 矩形・楕円・投げ縄選択と基本変形ができる
15. 対応端末で筆圧を描画へ反映できる
16. ページ・レイヤー増加時もOBS表示が安定する
17. ワークスペース全体をアーカイブで入出力できる
